如何正确的使用hexo+github+next部署自己的博客

费劲巴力我的博客总算是搭建起来了,本来想着自己动手写一个博客网站来的,但是能力有限,写出来的也没有人家模板的好看,不如用hexo一站式服务搭建自己的博客,所以第一篇文章就来记录如何搭建这样的一个博客.

安装git

  • git下载地址
  • 安装步骤：双击下载好的exe文件，一路next就好啦

安装nodejs

  • nodejs下载地址
  • LTS为长期支持版，Current为当前最新版(我们选长期支持版就好啦)
  • 双击打开安装，也是一路next，不过在Custom Setup这一步记得选 Add to path 这样就不用自己配置环境变量了
  • 安装好以后 win+r 打开 cmd 运行node -v 检验是否安装成功

    安装hexo

  • 安装hexo需要使用命令行 cmd, gitbash 均可,
  • 首先创建一个文件夹,放置所有博客的相关文件,并在该文件夹中打开命令行
  • 输入命令 npm i -g hexo 安装hexo
  • 安装完成以后输入hexo -v查看版本
  • 初始化命令 hexo init 初始化完成后就会看到该项目的文件夹辣
  • 每个文件夹的作用
    • node_models (依赖的模块包)
    • public：存放的是生成的页面
    • scaffolds：命令生成文章等的模板
    • source：用命令创建的各种文章
    • themes：主题
    • _config.yml：整个博客的配置
    • db.json：source解析所得到的
    • package.json：项目所需模块项目的配置信息
  • 这样你的博客的雏形就已经搭建完毕啦~

部署到github

  • 首先你需要有一个github的账号,我猜你一定有一个
    • 创建一个repo名字为你的github名称+github.io一定要严格遵循这个格式,否则将无效.
    • 创建ssh: 在命令杭中输入ssh-keygen -t rsa -C "youremail@example.com,替换邮箱为你的github邮箱,一路回车
    • 在C://User//yourname//.ssh中会发现几个文件,其中yourname为你的电脑用户名
    • 用记事本打开id_rsa.pub文件复制所有的内容
    • 打开你的github右上角头像下面点击setting
    • 点击ssh and gpg keys 菜单 然后点 new sshkey
    • title随便起一个,把刚才复制的一大串加入到key中点击保存即可完成
    • 在命令行中输入ssh -T git@github.com输入yes,验证是否成功,成功后如图所示
    • 用编辑器你打开你的blog项目,推荐vscode,提供了markdown编辑插件,而且可以在编辑器内使用命令行,方便的很(vscode天下第一)
    • 修改_config.yml文件的一些配置(冒号之后都是有一个半角空格的)
