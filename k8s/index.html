前言

k8s的前世今生

大家知道当前最火爆的容器管理技术就是docker,如果有不了解docker的可以看我之前的文章docker基础教程

在最开始搭建服务集群时,如果直接选择在物理机或者云服务器中搭建服务是不难的,我们直接通过对应的ip+端口访问即可映射,但是一旦选择了容器化以后互相的访问变得困难,需要进行端口映射 或者选择通道,会变的比较麻烦,还有我们对应服务的版本和回滚等等功能,为了方便管理,资源管理器应运而生

  • Apache mesos 是Apache基金会下的项目,最开始由美国的Twitter公司最先使用,但mesos最开始并不是用来管理docker容器的 所以Twitter公司在2019年5月选择使用了k8s,但现在docker火爆,mesos最新版中宣布可以在mesos平台中管理k8s,但是否好用就不得而知了

  • docker swarm 是docker的母公司开发的docker集群管理的功能,在新版本的docker中默认安装,可以说是十分轻量的一个资源,占用资源仅几十兆,而且经测试可以支持几千台的集群操控,然而docker swarm的功能实在是太少,不足以支持企业级的使用

    kubernetes 背书google公司,google是很早就开始使用容器化的一家公司,有十年容器化架构基础,于是基于go语言开发了k8s,有如下特点

    • 轻量级 资源消耗小
    • 开源 经济
    • 弹性伸缩
    • 负载均衡: IPVS

kubernetes的组件

  • apiserver: 所有服务访问统一入口
  • controller-manager:维持副本期望数目
  • schedule: 负责接受任务,选择合适的节点分配任务
  • ETCD: 简直对数据库,存储k8s集群的所有重要信息(持久化)
  • kubelet: 直接跟容器引擎交互实现容器的生命周期管理
  • kube-proxy: 负责写入规则至iptables IPVS实现服务映射
  • coreDNS: 可以为集群中的svc创建一个A记录
  • dashboard: 给k8s集群提供一个b/s结构的访问体系,即webUI
  • ingress-controller: 官方集群只能实现四层代理,ingress可以实现七层代理
  • Prometheus: 提供一个k8s集群的日志统一分析接入平台
